## Premium Bypass 报告模板

如果发现 premium/付费内容的客户端门控，额外生成此报告。结构与逆向报告相同（9 章），但聚焦于：

1. **门控机制**：客户端用了什么条件判断（如 `isRecharged`、`isPremium`）
2. **绕过方法**：至少提供 2 种方法（浏览器拦截 + 直接 API + API Key）
3. **服务端校验**：服务端是否有独立的二次检查

## 证据锚点格式规范

| 置信度 | 条件 |
|--------|------|
| HIGH | 源码中直接可见 + 动态调试/HAR 确认 |
| MEDIUM | 源码推断 + 未动态验证 |
| LOW | 仅 HAR 抓包推测，未找到源码对应 |

证据锚点格式：
- JS chunk: `<chunk-hash>.js` → module `<number>` → function `<name>`
- HAR: `HAR → <url> → request/response body`
- Network: `DevTools Network → <url> → <header/body field>`
