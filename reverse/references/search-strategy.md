# 搜索策略

从目标前端代码中提取关键信息的搜索清单。按优先级排序。

## 优先级 HIGH — 必须找到

| 搜索词 | 目的 | 找到后的操作 |
|--------|------|------------|
| `AES.encrypt` | 定位 AES 加密调用 | 回溯 key 变量 → 提取密钥 |
| `signMessage` | 定位签名逻辑 | 读签名消息构造 → 还原格式 |
| `encrypt` | Token 生成逻辑 | 识别加密算法和密钥来源 |
| `.sign(` | 签名调用点 | 确认签名方法和参数 |

## 优先级 MEDIUM — 补充信息

| 搜索词 | 目的 | 找到后的操作 |
|--------|------|------------|
| `localStorage` | 持久化的登录态 | 找出存储的 key 和值 |
| `sessionStorage` | 会话级存储 | 同上 |
| `fetch(` / `axios(` | 请求发送点 | 追踪端点和请求体格式 |
| `base64` | 可能有 base64 编码的密钥 | 检查长 base64 字符串周围逻辑 |
| `Authorization` | 认证头 | 确认 token 类型和格式 |
| `XOR` / `xor` | XOR 编码 | 找到 XOR key |

## 优先级 LOW — 辅助验证

| 搜索词 | 目的 | 找到后的操作 |
|--------|------|------------|
| `error` / `catch` | 错误处理逻辑 | 理解失败原因、错误码含义 |
| `rate.limit` / `throttle` | 速率限制 | 判断防刷策略 |
| `debug` / `console.log` | 调试残留 | 可能有额外的运行时信息 |
| `PRIVATE_KEY` / `API_KEY` / `SECRET` | 硬编码凭据 | 直接提取 |

## 搜索技巧

1. **先搜字符串，后搜符号**：`"Chain ID"` 比 `buildMessage` 更容易定位到关键函数
2. **注意消息格式模板**：模板字符串中的换行 (`\n`) 和缩进是签名消息的一部分，必须精确还原
3. **区分登录签名和业务签名**：同一个应用可能有多个签名消息格式（登录用一套，领取奖励用另一套）
4. **关注时间戳来源**：nonce/expiry 是从服务端获取还是本地生成，决定了重放的难度
